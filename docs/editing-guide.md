# Инструкция по обновлению данных портала

Файл предназначен только для редакторов и хранится в репозитории, а не на публичных страницах.

## Каталог ТОСов (`data/tos.data.js`)
1. Каждая запись — объект с `id`, названием, местоположением, контактами, ссылками и описанием.
2. `id` должен совпадать со слагом, который используется на карте и в ссылках.
3. После правок проверьте, что телефоны, ссылки VK/OK и количество жителей заполнены без лишних пробелов.

## Новости (`data/news.data.js`)
1. `type` принимает значения `news` или `announcement` — они попадают в фильтр на странице.
2. `content_html` поддерживает базовый HTML (абзацы, ссылки, списки).
3. Если есть источник, заполните `source_name` и `source_url`, иначе оставьте строку пустой.

## Документы (`data/docs.data.js`)
1. Все ссылки должны вести на реальные файлы. Предпочтительно размещать демо-файлы локально в `documents/demo/`.
2. Используйте короткие `id` без пробелов (например, `charter-demo`).
3. Поле `kind` отвечает за фильтрацию (например, «Шаблон», «Отчёт»).

## Карта (`data/tos.geo.js`)
1. Файл содержит `FeatureCollection`. В `properties` обязательны `id` (совпадает с `data/tos.data.js`), `name` и `place`.
2. Геометрию можно нарисовать в [geojson.io](https://geojson.io) и вставить в файл.
3. После обновления проверьте, что карта открывается и маркеры кликабельны.

## Проверка перед публикацией
- Откройте `index.html`, разделы «Каталог», «Новости», «Документы», «Карта» — ссылки не должны вести на заглушки.
- Убедитесь, что демо-материалы скачиваются, а инструкции не отображаются на пользовательских страницах.
